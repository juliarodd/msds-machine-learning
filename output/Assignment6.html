Background¶

MNIST ("Modified National Institute of Standards and Technology") is a dataset comprised of handwritten images of digits from zero through nine. The goal of this assignment is to correctly identify the digit of each image. Therefore, this is a multiclass classification problem.

The MNIST data set is comprised of 42,000 digits and 784 columns (there is an additional column for the digit classification). Each image is 28 pixels in height and 28 pixels in width, for a total of 784 pixels, which corresponds to the 784 columns. The data within each column represents the lightness or darkness of each pixel.

In this assignment, Python scikit-learn is used to build several different neural networks. While Tensor Flow was considered for this assignment, there were OSX runtime errors upon installation. A 2 x 2 design is used to build different models with run times and accuracy scores reported. Results are submitted to Kaggle for scoring. Commentary is provided at the end on which hyperparameter settings are most trustworthy.

Data Preparation¶

To prepare the data for modeling, we first standardize the data. This step is extremely important for neural networks, as it ensures that no extra bias is introduced into the model building process and it helps speed up processing time. Moreover, by standardizing the data, we can also take advantage of regularization techniques and ensure we get stable results.

In utilizing PCA techniques, it also important to standardize the data beforehand. Since PCA is a variable reduction technique, it transforms variables in an attempt to maximize variance. If standardization were not done, then variables with a higher variance would skew the PCA rotation/transformation process.

From the PCA output, we can see that by retaining 95% of data variance, we are able to reduce the number of variables from 784 to 320.

def confusion_matrix_heatmap(pred):
    cm = confusion_matrix(y_train, pred)
    cm = cm / cm.astype(np.float).sum(axis=1) #need to standardize the data first
    sns.heatmap(cm, vmin=0, vmax=1, cmap="Blues")

# make sure predictions are in right format
def pred_format(pred):
    pred = pd.DataFrame(pred, columns = ["Label"])
    pred['ImageId'] = range(1, len(pred) + 1)
    cols = pred.columns.tolist()
    cols = cols[-1:] + cols[:-1]
    pred_clean = pred[cols]
    return(pred_clean)

Summary and Conclusions¶

From the results above, models with one versus two hidden layers were compared to one another in addition to scaling the number of nodes from a small (20) to large (200) value. While the model with one layer and 200 nodes was the best-performing model, the model with two layers and 200 nodes was close behind it.

Depending on the complexity of the business problem/data set, the results above help to show that:

  • A large number of nodes helps to increase model performance.
  • Invoking the alpha parameter and early stopping are good measures to consider to help protect against overfitting.
  • Alternatively, PCA could be used to protect against overfitting. This method has the advantage of faster processing time.
  • More time/experimentation is needed to determine the optimal number of hidden layers.
  • Models with more hidden layers have comparable performance to models with one hidden layer. In other words, neural networks scale fairly well.
  • In general, neural networks are a powerful modeling method but ample time is needed to build an optimal model that does not overfit the data.
