Background¶

This analysis builds upon previous results, utilizing the Boston Housing data to advise a real estate brokerage firm on market value. Using all available variables, except neighborhood, this analysis explores predicting home value through various regression and tree-based methods. Ultimately, one final model is chosen and recommended to the real estate brokerage firm, with commentary provided on model results and important variables.

#calculate GOF metrics as a summary
def gof_sum(yhat, y, model_name, main_title):
    rsquared = round(r2_score(y, yhat),3)
    mse = mean_squared_error(y, yhat)
    rmse = round(sqrt(mse),3)
    mse = round(mse,3)
    mae = round(mean_absolute_error(y, yhat),3)
    
    x = PrettyTable()
    x.field_names = ["Model", "R-Squared", "MSE", "RMSE", "MAE"]
    x.add_row([model_name, rsquared, mse, rmse, mae])
    print(x.get_string(title=main_title))

#function for alpha tuning
def alpha_tuning(model):
    alphas = [0.0001, 0.001, 0.01, 0.1, 1, 10]
    val_errors = []
    for alpha in alphas:
        m_minmax = model(alpha=alpha, fit_intercept=True, random_state=RANDOM_SEED)
        errors = np.sum(-cross_val_score(m_minmax, 
                                     X_train, 
                                     y_train, 
                                     scoring='neg_mean_squared_error', 
                                     cv=10,
                                     n_jobs=-1))
        val_errors.append(np.sqrt(errors))
    print('best alpha: {}'.format(alphas[np.argmin(val_errors)]))

From the goodness of fit statistics table above, we can see that the R-squared value is fairly high at .745. However, we can see that there are differences in MAE and RMSE. For background, MAE is the average magnitude of errors, while the RMSE takes the square root of the squared difference between the residuals and fitted values. Therefore, RMSE gives more weight to values that are farther away from their actual value. If MAE and RMSE are close together or about equal, then we know that all the errors are of the same magnitude.

In our case, MAE and RMSE are not equal, which tells us that not all errors are of the same magnitude. As a step to improve model fit, residuals could be examined in greater detail to determine if there are influential points that should be removed.

Random Forest¶

In this section, we create a model using a random forest. A random forest is a collection of decision trees that injects randomness in the tree-building process. This randomness occurs by varying the variables considered in each tree and varying the data included to build the tree. By generating multiple random trees, this model is able to better discern nuances within the data and protect against overfitting.

We build the random forest on the raw boston housing dataset, as variable standardization has no impact on model performance.

One of the critical components of the random forest model is the hyperparameters used to build the tree. For instance, one hyperparameter involves defining the number of trees in the forest. Having too few trees can be disadvantageous, but generally there is a point where increasing the number of trees does not impact model performance. A few other key hyperparameters are included in scope for tuning, such as tree depth, the number of variables each tree can consider at each split, and how the model evaluates the best split.

This parameter grid will help to inform the best model to use for the random forest. Similar to the alpha tuning for the regression models above, the best parameters are selected on the basis of mean RMSE. Gradient Tree Boosting¶

In this section, we will utilize another variant of decision trees called gradient tree boosting. In general, the idea behind boosting is to combine several week learners together into a strong learner. Thus, with gradient boosting, the algorithm works by fitting several decision trees sequentially. Each decision tree fits a new predictor to the residual errors made by the previous predictor, trying to improve upon the overall predictions made. The final predictions are based on the sum of the predictions of all trees.

Similar to the random forest model building process, here we will utilize a parameter grid to tune different hyperparameters for this model. In addition to the normal tree hyperparameters, one other key hyperparameter is the learning rate. The learning rate for gradient tree boosting is akin to the alpha regularization parameter in ridge and lasso regression in that it helps to protect against overfitting by constraining how much each new tree can correct for the previous tree's errors.

Furthermore, we can see that the linear regression model outperforms the lasso and ridge regression models. This result is surprising. Because all variables have a small/medium effect on model performance, it seems like shrinking this impact from the ridge model has a worse impact on unseen data. With lasso, because there are no few predictors that greatly impact median home value, this model does not do as well.

Going a little bit deeper, we can see that the Gradient Tree Boosting model is the best model, as it has the highest R-squared value and the lowest MSE, RMSE, and MAE results. The RMSE results help to show how far away the predicted values are from the actual values (i.e., standard deviation) and are in the same units as median home value. Therefore, RMSE is an excellent measure to use not only for evaluating a model but for determining business goals around a final model. Here, we see that the Gradient Tree Boosting model is about $2,000 off from the actual values.

Summary and Conclusions¶

This analysis generated three different linear regression models and two tree-based models to predict median home value from the Boston housing data set. Both tree-based models ranked the variables of average rooms (rooms) and socioeconomic status (lstat) as important indicators for predicting median home value. The variables of location on the Charles River (chas), percent of land zoned for lots (zn), and access to radial highways (rad) appeared to have less impact on median home value, as they had low relative variable importance.

Moreover, the tree-based models outperformed the regression models when examining test set metrics. Specifically, the gradient tree boosting model performed the best. Therefore, tree-based methods should be used in the future to complement more traditional models. The decision of random forest vs gradient tree boosting is dependent on business goals. Random forests have greater explanatory power and good performance, but the gradient tree boosting model is more of a 'black box' process but has the best explanatory power. Better understanding business goals, especially if there are certain RMSE results that need to be achieved, would help to ensure that the best tree-based model is used.

To help improve the fit of the model, there are several steps that could be considered:

  • Inspect outliers and influential points. All observations were considered in this analysis, yet certain median home values may be outside the desired business range and if removed, could improve the fit of the final model. For instance, we saw above that the final model does a poor job of predicting higher (>40) median home values because there are so few of them. It may make sense to remove these homes altogether.
  • Continue to test different tree-based models. While some hyperparameter tuning was done, even more hyperparameter tuning could be done to ensure that the final hyperparameters chosen are the best. Additionally, comparing models that remove unimportant predictors would be a helpful step to ensure that all predictors are needed to have the best predictions.
  • Better understand business goals and ensure that the final model selected meets the business goals for explainability and predictive power.
